## 安装与准备

### 系统要求

- **操作系统：** Windows / macOS / Linux 均支持
- **Node.js：** 18 或更高版本
- **浏览器：** 系统已安装 Chrome

### 1. 安装 Node.js

前往 https://nodejs.org/ 下载安装。验证版本：

```bash
node -v
# 应输出 v18.x.x 或更高
```

### 2. 安装项目依赖

```bash
cd md-publisher
npm install
```

### 3. 安装浏览器

工具使用系统 Chrome 浏览器进行自动化操作。确保电脑已安装 Chrome，然后执行：

```bash
npx playwright install chromium
```

安装完成后即可使用。

## 文章格式

文章为标准 Markdown 文件，文件开头使用 YAML frontmatter 声明元数据：

```markdown
---
title: 文章标题
shortTitle: 短标题（可选）
description: 文章摘要（可选）
date: 2026-01-20
category:
  - 分类一
  - 分类二
tag:
  - 标签一
  - 标签二
---

这里是正文内容，使用标准 Markdown 语法...
```

### 字段说明

| 字段          | 必填 | 说明                                                                         |
| ------------- | ---- | ---------------------------------------------------------------------------- |
| `title`       | 是   | 文章标题，自动填入各平台标题栏                                               |
| `shortTitle`  | 否   | 短标题，部分平台使用                                                         |
| `description` | 否   | 文章摘要，微信公众号会自动填入摘要栏                                         |
| `date`        | 否   | 发布日期                                                                     |
| `category`    | 否   | 分类，字符串或数组格式。掘金和什么值得买会自动选择                           |
| `tag`         | 否   | 标签，字符串或数组格式。掘金（最多 3 个）和什么值得买（最多 5 个）会自动填写 |

### 分类/标签优先级

当文章 frontmatter 和项目配置（`.md-publisher.yml`）都指定了 `category` 或 `tag` 时：

- **文章 frontmatter 优先** —— 如果文章自身有 `category`/`tag`，使用文章的
- **项目配置兜底** —— 如果文章没有指定，回退到项目配置中的值

这样可以在项目配置中设置通用的分类/标签，同时允许个别文章覆盖。

### 支持的文件类型

`.md` `.markdown` `.mdown` `.mkd` `.mkdn` `.mdwn` `.mdx` `.txt`

### 自动跳过的文件

扫描目录时会自动跳过以下文件：`README.md`、`CHANGELOG.md`、`LICENSE.md`、`LICENSE.txt`、`.md-publisher.yml`，以及 `node_modules/`、`user-data/` 目录和所有以 `.` 开头的隐藏文件/目录。

## 项目配置

在文章目录下创建 `.md-publisher.yml`（或 `.md-publisher.yaml`）文件，可为该目录下的所有文章预设发布参数。CLI 和 GUI 都会自动读取该配置。

### 配置示例

```yaml
# 预选的发布平台（覆盖默认勾选）
platforms:
  - sspai
  - zhihu
  - juejin

# 文章分类（传递给支持分类的平台：掘金、什么值得买）
category:
  - 技术
  - 效率

# 文章标签（掘金最多 3 个，什么值得买最多 5 个）
tag:
  - Markdown
  - 写作工具
```

### 配置字段

| 字段         | 类型         | 说明                                                                                    |
| ------------ | ------------ | --------------------------------------------------------------------------------------- |
| `platforms`  | 数组         | 预选的平台标识列表，覆盖默认勾选。可选值：`sspai` `zhihu` `wechat` `smzdm` `juejin` |
| `category`   | 字符串或数组 | 文章分类，作为各文章的默认分类（文章自身的 frontmatter 优先）                           |
| `tag`        | 字符串或数组 | 文章标签，作为各文章的默认标签（文章自身的 frontmatter 优先）                           |

### 使用方式

- **CLI 模式：** 检测到配置后，终端会显示配置概要（平台、分类、标签等）。平台选择默认勾选会被配置覆盖
- **GUI 模式：** 加载文章目录时自动读取配置，界面上方会显示项目配置标识

> 没有 `.md-publisher.yml` 时使用内置默认值（默认平台：少数派、知乎、公众号），不影响正常使用。

## CLI 命令行模式

适合熟悉命令行的用户，交互式逐步操作。

### 基本用法

```bash
node src/index.js [path]
```

`path` 可选，支持三种用法：

| 用法     | 命令                           | 说明                             |
| -------- | ------------------------------ | -------------------------------- |
| 无参数   | `npm start`                    | 递归扫描当前目录下的所有文章文件 |
| 指定文件 | `node src/index.js article.md` | 直接使用该文章                   |
| 指定目录 | `node src/index.js ./posts/`   | 递归扫描该目录下的所有文章文件   |

### 操作流程总结

```
┌─────────────────────────────────────────────────────┐
│  1. 选择文章                                         │
│     ├─ 参数为文件 → 直接使用                          │
│     ├─ 参数为目录 / 无参数 → 扫描后弹出勾选列表       │
│     └─ 目录为空 → 提示手动输入路径                    │
│                                                     │
│  2. 显示项目配置（如有 .md-publisher.yml）             │
│                                                     │
│  3. 选择平台（空格勾选，回车确认）                     │
│     └─ 默认勾选：少数派、知乎、公众号                  │

│                                                     │
│  4. 确认发布                                         │
│                                                     │
│  5. 自动执行：逐篇文章 × 逐个平台                     │
│     ├─ 打开平台编辑器                                │
│     ├─ 检测登录（未登录则提示并等待）                  │
│     ├─ 填写标题、粘贴内容                            │
│     └─ 填写分类/标签（掘金、什么值得买）              │
│                                                     │
│  6. 显示结果汇总表                                    │
│                                                     │
│  7. 浏览器保持打开，用户检查后按 Enter 关闭            │
└─────────────────────────────────────────────────────┘
```

## GUI 图形界面模式

提供 Web 图形界面，操作更直观，适合日常使用。

### 启动方式

```bash
npm run gui
```

启动后自动打开浏览器访问 `http://localhost:9870`。

也可以使用快捷启动脚本（日志输出到 `md-publisher.log`）：

- **Windows：** 双击 `start.bat` 或 `start.vbs`（静默启动，无黑窗）
- **macOS：** 双击 `start.command`（终端窗口自动关闭）
- **Linux：** 运行 `chmod +x start.sh && ./start.sh`（后台启动，显示 PID）

### 环境变量

| 变量      | 说明                      | 默认值 |
| --------- | ------------------------- | ------ |
| `PORT`    | 自定义端口号              | `9870` |
| `NO_OPEN` | 设为 `1` 不自动打开浏览器 | —      |

```bash
# 示例：使用 8080 端口，不自动打开浏览器
PORT=8080 NO_OPEN=1 node src/gui.js
```

### 界面功能

#### 1. 加载文章

两种方式添加文章：

- **扫描目录**：在设置中填写文章目录路径，工具自动递归扫描目录下的所有 Markdown 文件，按修改时间倒序排列。如果目录下有 `.md-publisher.yml` 配置文件，界面会显示项目配置标识
- **手动添加**：点击「选择文件」或「选择目录」按钮上传文章文件（最多 100 个文件，单文件不超过 10MB，仅接受支持的扩展名）。也可以在输入框中直接输入文件/目录路径后按 Enter 添加

加载后的文章显示标题、文件名和摘要信息，可以逐篇勾选要发布的文章，也可以使用「全选」「取消全选」按钮批量操作。

#### 2. 选择平台

勾选目标平台。**默认勾选：** 少数派、知乎、公众号（可在设置中自定义）。如果加载的文章目录含有 `.md-publisher.yml` 项目配置，则使用配置中的平台。

#### 3. 发布与取消

点击「发布」按钮后：

- 界面实时显示发布进度（正在处理哪篇文章、哪个平台）
- 日志面板实时滚动输出操作详情
- 每个平台完成后显示成功/失败状态
- 发布过程中按钮置灰，防止重复提交
- **随时取消**：进度条旁会出现「取消」按钮，点击后安全中断当前发布流程

发布完成后，自动化 Chrome 窗口中的各平台标签页保持打开，你可以在其中检查内容并手动点击各平台的「发布」按钮。

#### 4. 检测登录状态

点击「检查登录」按钮，工具在独立窗口中并行打开所有平台页面检查登录状态，结果实时更新到界面上，显示每个平台的登录状态和用户名。检测结果会保存到 `settings.json`，下次打开时直接显示。

如果某个平台显示未登录，可以在自动化 Chrome 中手动访问该平台完成登录后再次检测。

#### 5. 关闭发布窗口

点击「关闭发布窗口」按钮可关闭 Playwright 控制的自动化 Chrome 窗口，释放系统资源。

**注意：** 此操作仅关闭后台的自动化 Chrome 实例，不会影响你正在操作的 GUI 页面。下次发布时工具会自动重新启动浏览器。

#### 6. 退出应用

点击页面右上角的「退出」按钮可关闭整个 md-publisher 服务（包括自动化浏览器和 Web 服务器）。退出前会弹出确认对话框。

#### 7. 设置

- **文章目录**：设置默认扫描的文章目录路径。可以直接输入路径，也可以点击「浏览」按钮在目录树中选择
- **默认平台**：在发布页勾选/取消平台时自动保存为默认选择

设置自动保存到项目根目录的 `settings.json` 文件。

### CLI 与 GUI 功能对比

| 功能     | CLI            | GUI                            |
| -------- | -------------- | ------------------------------ |
| 文章选择 | 交互式勾选列表 | 目录扫描 + 文件上传 + 手动添加 |
| 平台选择 | 交互式勾选列表 | 可视化勾选                     |
| 项目配置 | 自动读取显示   | 自动读取，界面显示配置标识     |
| 发布进度 | 终端实时输出   | SSE 实时推送 + 进度条          |
| 取消发布 | Ctrl+C 中断    | 取消按钮                       |
| 登录检测 | 发布时自动检测 | 独立检测按钮，并行检查所有平台 |
| 结果汇总 | 终端表格       | 界面状态标记                   |

## 各平台发布详解

### 少数派 (sspai)

| 项目       | 说明                                                 |
| ---------- | ---------------------------------------------------- |
| 编辑器     | CKEditor 5                                           |
| 自动操作   | 填写标题 → 粘贴 Markdown → 点击「立即转换」按钮      |
| 内容处理   | 自动去除图片 URL 中的 `?imageMogr2/format/webp` 参数 |
| 分类/标签  | 不支持自动填写（有设置时会提示）                     |
| 需手动完成 | 检查转换后的排版，点击发布                           |

工具打开少数派编辑页面，自动填入标题和正文。粘贴 Markdown 后会自动点击编辑器弹出的「立即转换」按钮，将 Markdown 转为富文本格式。

### 知乎 (zhihu)

| 项目       | 说明                                              |
| ---------- | ------------------------------------------------- |
| 编辑器     | Draft.js                                          |
| 自动操作   | 填写标题 → 粘贴 Markdown → 点击「确认并解析」通知 |
| 内容处理   | 自动去除图片行前后的空行（修复知乎解析排版问题）  |
| 分类/标签  | 不支持自动填写（有设置时会提示）                  |
| 需手动完成 | 检查解析后的内容，添加话题标签，点击发布          |

粘贴 Markdown 内容后，知乎会弹出瞬态通知栏询问是否解析 Markdown。工具采用预竞争策略：在粘贴之前就启动并行定位器等待通知出现，确保不会错过短暂弹出的通知，自动点击「确认并解析」。

### 微信公众号 (wechat)

| 项目       | 说明                                                                                        |
| ---------- | ------------------------------------------------------------------------------------------- |
| 编辑器     | ProseMirror（微信自带富文本编辑器）                                                         |
| 自动操作   | 打开转换器 → 粘贴 Markdown → 复制富文本 → 跳转微信编辑器 → 填写标题 → 粘贴富文本 → 填写摘要 |
| 内容处理   | 通过 md.doocs.org 将 Markdown 转为排版精美的富文本 HTML                                     |
| 分类/标签  | 不支持自动填写（有设置时会提示）                                                            |
| 需手动完成 | 设置封面图，检查排版，点击发布                                                              |

微信公众号不支持直接粘贴 Markdown，因此工具采用两步操作：

1. 先打开 md.doocs.org Markdown 转换工具，粘贴文章内容并点击「复制」按钮
2. 跳转到微信公众号编辑器，自动从 URL 提取登录 token 并导航到新建图文页面
3. 填入标题、粘贴富文本内容、填写摘要

### 什么值得买 (smzdm)

| 项目       | 说明                                                                  |
| ---------- | --------------------------------------------------------------------- |
| 编辑器     | ProseMirror                                                           |

## 登录管理

### 首次登录

首次使用某个平台时，工具会自动打开该平台页面并检测登录状态。如果未登录：

1. 多重提醒通知：
   - 终端提示音（BEL）
   - 彩色终端输出 + 标题栏闪烁
   - **Windows：** 系统托盘气泡通知
   - **macOS：** 系统通知中心 + 提示音
   - **Linux：** `notify-send` 桌面通知
2. 在自动打开的浏览器中手动完成登录
3. 登录成功后工具自动检测并继续操作（也可按 Enter 手动确认）

登录等待超时时间为 5 分钟，足够完成各种验证方式。

### 登录状态持久化

登录状态保存在项目 `user-data/` 目录中（Chrome 用户数据目录），关闭工具后登录态不会丢失。下次启动时直接使用已有的登录状态，无需重复登录。

### 批量检测登录状态（GUI）

在 GUI 界面点击「检查登录」按钮，工具会在独立窗口中并行打开所有平台页面，几秒内完成检测并显示每个平台的登录状态和用户名。检测结果会保存到 `settings.json`，下次打开页面时直接显示。

如果某个平台显示未登录，可以在自动化浏览器中手动访问该平台完成登录后再次检测。

## 常见问题

### 端口被占用

```
❌ 端口 9870 已被占用
```

说明已有一个 md-publisher 实例在运行。关闭它后重试，或使用其他端口：

```bash
PORT=8080 npm run gui
```

### 浏览器未启动

确保已安装 Chrome 浏览器和 Playwright chromium：

```bash
npx playwright install chromium
```

### 剪贴板写入失败

```
❌ 剪贴板写入失败，请检查浏览器权限
```

工具使用浏览器 Clipboard API 进行粘贴操作。如果出现此错误，可能是浏览器未授予剪贴板权限。工具启动时已自动请求 `clipboard-read` 和 `clipboard-write` 权限，通常不会出现此问题。如果仍然报错，尝试重启工具或手动在 Chrome 设置中允许剪贴板访问。

### 粘贴内容为空或不完整

部分较慢的系统可能需要更长的剪贴板操作等待时间。可以在 `src/config.js` 中调大 `TIMEOUTS` 中的粘贴相关参数：

```javascript
pasteBeforeKeys: 200,   // 点击后、按键前的等待时间
pasteBetweenKeys: 100,  // Ctrl+A/Cmd+A 和 Ctrl+V/Cmd+V 之间的等待时间
pasteAfterKeys: 300,    // Ctrl+V/Cmd+V 后等待编辑器处理的时间
```

> 工具会自动识别操作系统，macOS 上使用 Cmd 键，Windows/Linux 上使用 Ctrl 键。

### 浏览器连接断开

如果手动关闭了自动化 Chrome 窗口，工具会自动检测并在下次操作时重新启动。

### 发布任务冲突

```
已有发布任务正在进行
```

同一时间只能运行一个发布任务。等待当前任务完成（或在 GUI 中点击取消）后再发起新任务。

### 重置登录状态

如果登录状态异常（如 Cookie 过期导致反复提示登录），可以删除 `user-data/` 目录来清除所有保存的登录数据：

```bash
rm -rf user-data/
```

下次启动时需要重新登录所有平台。

### 查看运行日志

使用启动脚本启动时，日志输出到项目根目录的 `md-publisher.log` 文件。可以查看此文件排查问题：

```bash
# 实时查看日志
tail -f md-publisher.log

# Windows PowerShell
Get-Content md-publisher.log -Wait
```

### 文章扫描不到文件

检查以下几点：

- 文件扩展名是否为支持的类型（`.md` `.markdown` `.mdown` `.mkd` `.mkdn` `.mdwn` `.mdx` `.txt`）
- 文件是否在隐藏目录（以 `.` 开头）中
- 文件名是否为被跳过的类型（`README.md`、`CHANGELOG.md`、`LICENSE.md`、`LICENSE.txt`）
- 文件是否在 `node_modules/` 或 `user-data/` 目录中

### 分类/标签未自动填写

目前只有**掘金**和**什么值得买**支持自动填写分类和标签。其他平台（少数派、知乎、公众号）会在日志中提示「暂不支持自动填写分类/标签」，需要手动操作。

分类/标签的来源优先级：**文章 frontmatter > 项目配置 (.md-publisher.yml) > 无**。确保在文章或项目配置中设置了 `category`/`tag` 字段。
